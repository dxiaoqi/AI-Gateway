# Iteration 16：配额策略管理

- 版本：0.16.0（合并交付版本 0.18.0）
- 日期：2026-07-15
- 状态：Completed
- 关联 ADR：[ADR 0015](../adr/0015-governance-resources-and-runtime-enforcement.md)

## 0. 一句话说明

管理员可以在后台发布租户、项目、应用和 Key 四级配额，请求会立即执行新规则。

## 1. 30 秒领导摘要

| 问题 | 回答 |
|---|---|
| 以前 | 限流规则写在启动配置，业务团队不能自助调整 |
| 现在 | 后台管理 RPM、TPM、最大并发，按租户隔离并即时参与 Redis/内存预占 |
| 价值 | 防止单个团队挤占共享模型容量，变更可追踪、可回滚 |
| 证据 | 自动化测试发布 RPM=1 后，第一个正常请求成功、第二个返回 429 |
| 风险 | 尚无策略审批和定时生效窗口 |

## 2. 小白导读

RPM 是一分钟最多多少次请求，TPM 是一分钟最多处理多少 Token，并发是同一时刻最多几次尚未结束的调用。它们分别类似前端的接口频率限制、上传总大小限制和请求连接池大小。

请求流程：身份认证得到 tenant/project/application/key → 读取该租户启用策略 → 与环境变量中的基础策略合并（同 ID 时后台策略覆盖）→ 原子预占 → 调模型 → 按实际 Token 结算；失败则取消预占。

## 3. 详细设计

- `scope + scopeId` 决定规则匹配对象。
- 至少配置 RPM、TPM、并发中的一个，所有数值必须大于零。
- 动态策略复用现有 `QuotaService` 和 Redis Lua 原子语义，没有另造限流器。
- 环境变量策略仍兼容；动态策略同 ID 时覆盖，避免重复扣减。
- 后台 viewer 只读，operator/admin 可发布；跨租户创建与更新返回 403。

预期：护栏阻断请求不会消耗 RPM；普通请求第一次 200，第二次 `quota_requests_exceeded`。

## 5. 已知限制

- 没有草稿、审批、未来时间发布和策略模拟器。
- 策略读取当前逐请求访问 PostgreSQL；高吞吐生产部署需要版本缓存与失效广播。
- Redis 不可用时的生产降级策略仍由现有限流运行模式决定。
