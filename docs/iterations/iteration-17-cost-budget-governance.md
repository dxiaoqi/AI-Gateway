# Iteration 17：成本与预算治理

- 版本：0.17.0（合并交付版本 0.18.0）
- 日期：2026-07-15
- 状态：Completed
- 关联 ADR：[ADR 0015](../adr/0015-governance-resources-and-runtime-enforcement.md)

## 0. 一句话说明

网关能按模型 Token 价格记账，并在租户月度预算耗尽后阻止继续产生供应商费用。

## 1. 30 秒领导摘要

| 问题 | 回答 |
|---|---|
| 以前 | 只知道 Token，无法回答“花了多少钱、还能不能继续调用” |
| 现在 | 后台配置模型输入/输出单价和租户月预算，成功调用自动聚合，超预算调用前拒绝 |
| 价值 | 把成本控制从月末看账单前移到请求发生前 |
| 证据 | 测试先产生大于预算的用量，下一次请求返回 `budget_exceeded` |
| 风险 | 并发检查和记账非单事务，可能小幅超额；尚未和供应商账单对账 |

## 2. 小白导读

模型通常按输入和输出 Token 分别定价。计算方式：

`费用 = 输入 Token × 输入百万单价 / 1,000,000 + 输出 Token × 输出百万单价 / 1,000,000`

这类似前端埋点：请求成功后上报用量；区别是这里会在下一次请求前读取累计金额，并执行硬性预算闸门。

## 3. 详细设计

- 定价规则按租户、逻辑模型、CNY/USD 配置，输入和输出价格都必须大于零。
- 当前支持自然月，`period` 格式为 `YYYY-MM`。
- 聚合表以租户、月份、币种为主键，使用数据库 upsert 累加金额和 Token。
- 预算检查发生在配额预占和供应商调用之前，超限返回不可重试 429。
- 记账故障写错误日志，但不会把已经成功的模型响应改成 500，避免客户重试造成二次费用。

## 4. 数据与隐私

用量表只保存租户、月份、币种、金额和 Token 数，不保存 Prompt、回复或虚拟 Key 明文。金额数据库列使用 `numeric(20,8)`。
