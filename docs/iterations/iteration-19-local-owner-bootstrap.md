# Iteration 19：主组织账号首次注册与密码登录

- 版本：0.19.0
- 日期：2026-07-15
- 状态：Completed
- 关联 ADR：[ADR 0016](../adr/0016-local-owner-bootstrap.md)

## 0. 一句话说明

没有 OIDC 时可以首次注册唯一主组织 Owner，之后使用用户名和密码登录管理后台。

## 1. 30 秒领导摘要

| 领导关心的问题 | 回答 |
|---|---|
| 以前的问题 | IdP 未接入时页面没有可用登录入口 |
| 本轮交付 | 首次注册、组织 Owner、密码强哈希、账号登录、短期令牌、失败限速和现有 Session 接入 |
| 业务价值 | 项目可立即演示和运营，同时保留以后切换企业 SSO 的路径 |
| 验证证据 | 自动化覆盖唯一注册、登录、限速、管理 API 和个人审计；浏览器验证注册入口可见 |
| 最大剩余风险 | 当前只有一个 Owner，不支持找回密码、MFA 和多人账号 |

## 2. 新人工程师导读

### 本轮术语

| 术语 | 小白解释 | 前端类比 |
|---|---|---|
| Bootstrap | 系统第一次创建管理员 | 应用首次安装向导 |
| Owner | 主组织最高管理员 | SaaS 工作区创建者 |
| scrypt | 把密码变成难以暴力破解的摘要 | 比普通 SHA 更慢、更适合密码 |
| Access Token | Gateway 能验证的短期身份证明 | 登录后的短期 API 凭证 |
| HttpOnly Session | 浏览器脚本读不到的会话 Cookie | 服务端 BFF Session |

### 一次注册发生了什么

1. 页面读取 Gateway 状态，只有账号文件不存在时展示“创建主组织”。
2. 用户填写组织名、用户名和强密码；Next.js 校验 Origin 后转发到 Gateway。
3. Gateway 校验字段，用 scrypt 生成盐和摘要，并以 `wx` 独占模式创建账号文件。
4. 如果两个注册同时到达，只有一个能创建文件，另一个得到 409。
5. Gateway 签发短期令牌；Next.js 把它放入服务端 Session，只把 HttpOnly Cookie 发给浏览器。
6. 页面用 Session 调用 `/admin/v1/me` 和其他管理 API，审计身份为 `local:<accountId>`。

## 3. 目标与非目标

目标：无需 OIDC 即可安全进入；公开注册只存在一次；密码不落明文；登录有暴力破解保护；复用 CSRF/BFF。

非目标：开放用户注册、多个组织、邀请成员、密码找回、MFA、社交登录。

## 4. 安全与错误语义

- 密码要求 12–128 位，至少包含字母和数字。
- 用户名不存在与密码错误使用同一 401 文案，减少账号枚举。
- 同 IP+用户名 15 分钟内失败 5 次后返回 429。
- Access Token 固定 HS256，并校验 issuer/audience/typ/exp，不能接受 OIDC 的非对称配置混用。
- 生产必须设置独立的签名密钥和 Bootstrap Token，均至少 32 字符。
- 账号文件目录权限 `0700`，文件 `0600`，且已加入 `.gitignore`。

## 5. 验证

```bash
npx vitest run test/local-admin-auth.test.ts
npm run verify
```

成功标准：首次注册 201、第二次注册 409、正确密码 200、错误密码 401、连续失败 429、令牌访问 `/admin/v1/me` 为 200、审计 `authMethod=local`。

## 6. 已知限制

- 无数据库时账号会持久化，但 Key、治理策略等控制面配置仍是进程内数据，重启会恢复环境种子；正式使用应配置 PostgreSQL。
- 单机账号文件不支持多实例，Kubernetes 部署应使用 OIDC，或等待数据库本地账号仓库。
- 唯一 Owner 无法演示双人 Key 轮换批准；需要 OIDC 多用户或后续成员邀请能力。
