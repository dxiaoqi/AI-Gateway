# 治理能力操作手册（Iteration 15–18）

## 给第一次接触项目的人

管理后台的四个新入口分别控制“调用谁、能调用多少、最多花多少、什么内容不能发出去”。修改点击保存后会直接影响网关请求，请先在测试租户验证。

## 启动前

```bash
npm run db:migrate
npm run dev
npm run admin:dev
```

访问 `http://127.0.0.1:3100`，登录后可以看到模型部署、配额策略、成本预算和安全护栏。

## 推荐操作顺序

1. 在“模型部署”创建供应商映射。外部供应商只填写凭证环境变量名，例如 `OPENAI_API_KEY`，不要粘贴密钥。
2. 在“配额策略”为测试租户设置较宽松的 RPM/TPM，先确认正常请求不被误限。
3. 在“模型定价”填写百万 Token 单价，再创建同币种月度预算。
4. 在“安全护栏”先使用 `audit` 观察，确认误报范围后再切到 `block`。

## 常见错误

| 错误码 | 意思 | 处理 |
|---|---|---|
| `precondition_required` | 更新没带版本号 | 刷新列表后用当前版本重试 |
| `resource_conflict` | 别人已经修改过 | 不要覆盖，刷新并核对变化 |
| `quota_*_exceeded` | 命中限流 | 检查策略范围和数值，必要时等待窗口恢复 |
| `budget_exceeded` | 本月聚合费用达到预算 | 核对用量与预算，批准后再提高上限 |
| `content_policy_violation` | Prompt 命中阻断护栏 | 删除敏感内容或由安全人员调整策略 |

## 紧急回滚

后台将对应资源切换为“停用”。停用模型部署会立即从路由表移除；停用配额、预算或护栏后，新请求不再执行该规则。停用动作也有版本保护和审计记录。

## 安全提醒

- 模型凭证必须通过进程环境或后续 Secret Manager 注入。
- Prompt 原文不会进入治理资源或用量表。
- 当前正则 Guardrail 只能作为基础防线，敏感生产场景仍需接入企业 DLP/内容审核服务。
- 预算检查与记账目前不是一笔数据库事务，极高并发下可能小幅超额。
